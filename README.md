# tg-assistance-bot

Telegram-бот на [pyTelegramBotAPI](https://github.com/eternnoir/pyTelegramBotAPI): список задач в PostgreSQL и ответы нейросети [GigaChat](https://developers.sber.ru/docs/ru/gigachat/overview) на произвольные сообщения (роль «секретарь» в промпте задаётся в коде).

## Возможности

- **Задачи**: добавление, просмотр, удаление, отметка выполненными; опциональный срок `execute_at` в БД.
- **Свободный текст** (только для пользователей из белого списка): запрос уходит в GigaChat; ответ модели показывается в чате.
- **Кнопки** reply-клавиатуры: «Добавить задачу», «Удалить задачу», «Список задач» (см. `controller/buttons.py`). Команда `/start` по умолчанию не подключает клавиатуру. Обработчик произвольного текста (GigaChat) в `controller/commands.py` срабатывает раньше обработчика кнопок, поэтому для задач стабильнее пользоваться командами `/add`, `/list`, `/delete`, `/complete`.

## Доступ

Разрешённые пользователи задаются в `auth.py`: множество `ALLOWED_TG_USERS` (username **без** `@`) и при необходимости `ALLOWED_TG_USER_IDS` (числовые id). Остальным бот отвечает «Нет доступа.» (кроме `/start`).

## Переменные окружения

| Переменная | Назначение |
|------------|------------|
| `BOT_TOKEN` | токен бота от [@BotFather](https://t.me/BotFather) |
| `GIGA_API_KEY` | ключ авторизации GigaChat |
| `PGHOST`, `PGDATABASE`, `PGUSER`, `PGPASSWORD`, `PGPORT` | подключение к PostgreSQL |

Файл `.env` подхватывается через `python-dotenv`.

## Установка и запуск

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

При старте выполняется проверка подключения к БД и при необходимости создаётся таблица `tasks` (см. `model/schema.py`).

## Команды (для пользователей из белого списка)

| Команда | Описание |
|---------|----------|
| `/start` | приветствие и версия бота |
| `/list` | показать нумерованный список задач (невыполненные и выполненные, сроки по возможности) |
| `/add …` | добавить задачу. Формат: `/add текст` или `/add текст execute_at дата-или-время` (маркер `execute_at` разделяет описание и срок) |
| `/delete N M …` | удалить задачи с номерами из последнего списка (цифры в сообщении) |
| `/complete N M …` | отметить задачи выполненными по номерам из списка |

## Кнопки

После нажатия **«Добавить задачу»** бот ждёт следующее сообщение: текст задачи или строка вида `текст execute_at дата` (как для `/add` без префикса). **«Удалить задачу»** — затем номера через пробел или запятую. **«Список задач»** — то же, что `/list`.

Если клавиатура не показана, все те же действия доступны через команды выше.

## Если бот не отвечает ожидаемо

- Проверьте, что ваш `username` или `id` есть в `auth.py`.
- Убедитесь, что заданы `BOT_TOKEN` и переменные PostgreSQL; в логах при старте пишется результат подключения к БД.
- Для ответов GigaChat нужен `GIGA_API_KEY`; при ошибке API в чат уйдёт сообщение «Ошибка AI: …».

## Структура репозитория

- `main.py` — точка входа, polling.
- `controller/` — обработчики команд, кнопок, вызов GigaChat, разбор текста команд.
- `model/` — PostgreSQL: схема, CRUD по задачам.
- `view/` — форматирование списка задач и inline-подтверждение добавления (колбэки «Да» / «Нет»).
- `auth.py` — белый список пользователей.

Версия приложения задаётся в `version.py`.
